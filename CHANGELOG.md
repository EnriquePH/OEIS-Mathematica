# Changelog

All notable changes to this project are documented here. The format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and versions match
`PacletInfo.wl` / the `:Package Version:` header in `Kernel/OEIS.wl`.

## [4.0.0] - 2026-07-21

### Added
- Converted the repository into an installable Wolfram Language paclet:
  `PacletInfo.wl`, `Kernel/init.m`, source moved to `Kernel/OEIS.wl`.
  `OEIS.m` at the repository root is kept as a thin backward-compatible
  loader, so `Get["OEIS.m"]` keeps working unchanged.
- `Tests/OfflineTests.wlt` and `Tests/NetworkTests.wlt`: a `VerificationTest`
  suite, runnable via `Tests/RunTests.wls` (`--offline` to skip the tests
  that need the live OEIS API).
- Generated reference documentation notebooks for every public symbol under
  `Documentation/English/ReferencePages/Symbols`, a guide page
  (`Documentation/English/Guides/OEIS.nb`) and a tutorial notebook
  (`Documentation/English/Tutorials/OEIS.nb`), all built by
  `DevTools/BuildDocumentation.wls` with real, live-executed example output.
- `docs/functions.md`: a standalone alphabetical function index.
- `Benchmarks/Benchmark.wls` and `Benchmarks/Benchmark.nb`: timing of the
  package's core operations, generated by
  `DevTools/BuildBenchmarkNotebook.wls`.
- GitHub Actions: `.github/workflows/ci.yml` (Linux/macOS/Windows test
  matrix), `.github/workflows/release.yml` (builds and attaches a `.paclet`
  archive to GitHub Releases), `.github/workflows/wolfram-version-compat.yml`
  (offline tests against multiple Wolfram Engine Docker images).
- `DevTools/BuildPaclet.wls`: builds the installable `.paclet` archive
  locally (`wolframscript -file DevTools/BuildPaclet.wls`).
- `Assets/Icon.png` / `Assets/Icon128.png`: package icon, referenced from
  `PacletInfo.wl`.
- `.zenodo.json` and an updated `CITATION.cff`, so a GitHub release can be
  archived on Zenodo with a DOI (see README for the one-time setup step).

### Fixed
- `OEISbFile` (originally shipped in 3.1): now uses a session-local
  `id[n_] := ...` definition, if present, to compute every term up to
  `VMax` -- previously it silently ignored that definition and truncated
  output to whatever OEIS already had published, regardless of `VMax`.

## [3.1] - 2026-07-11

### Fixed
- JSON result parsing: the live API returns a bare array, not a
  `{"results": [...]}` wrapper.
- Offset parsing for `"start,digits"`-style strings (e.g. `"0,4"`).
- BibTeX date parsing for ISO-8601 timestamps.
- Added a `curl`-based fallback for HTTP fetches when Mathematica's
  built-in client is unavailable (see `OEISReadJSONViaCurl`).

## [3.0] - 2026-07-11

### Changed
- Rebuilt around the official OEIS JSON API, replacing the earlier
  HTML-scraping implementation. Public function names kept for
  compatibility: `OEISTotalNumberOfSequences`, `OEISValidateIDQ`,
  `OEISImport`, `OEISURL`, `OEISFunction`, `OEISExport`, `OEISbFile`.

## [2.0] - 2010-11-28

### Changed
- Code rebuilt after upstream OEIS link changes broke the previous version.

## [1.0] - 2010-08-26

### Added
- Initial release: `OEISbFile` and HTML export.
